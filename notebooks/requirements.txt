1. Introduction and Problem Framing (Part A, Q1)
Selected Domain: Healthcare – Patient Intelligence
The analytical problem addressed is the prediction of diabetes onset risk in patients using clinical and demographic data.
Decision Objective
To reduce hospital readmissions and long-term complications by identifying high-risk patients early, allowing targeted preventive care like; lifestyle counselling and medication adjustment among others. This objective directly supports better resource allocation and improved patient outcomes in healthcare systems.
Relevant Variables
Structured: Pregnancies, Glucose, BloodPressure, SkinThickness, BMI, DiabetesPedigreeFunction, Age (target = Outcome: 0 = No diabetes, 1 = Diabetes based on probabilities, Threshold 0.4).
Textual: Medical abstracts and patient notes (for sentiment and keyword insights).
Role of Data-Driven Modelling
Machine learning models analyze historical patient data to generate risk scores that shift healthcare from reactive to proactive decision-making. These predictions integrate with electronic health records for real-time alerts.


2. Dataset Acquisition and Preparation (Part A, Q2 & Q3)
Dataset
1.	Structured data: Pima Indians Diabetes dataset (768 rows, 9 columns) acquired directly from the public URL:
https://raw.githubusercontent.com/jbrownlee/Datasets/master/pima-indians-diabetes.csv
2.	Textual data: Medical Abstracts TC Corpus (11,550 abstracts) acquired from:
https://raw.githubusercontent.com/sebischair/Medical-Abstracts-TC-Corpus/main/medical_tc_train.csv

Cleaning and Preparation Steps
•	Replaced biologically impossible zeros (Glucose, BloodPressure, SkinThickness, Insulin, BMI) with NaN and imputed using median.
•	Removed outliers using Inter-Quartile Range (IQR) method on all numeric features.
•	Cleaned dataset saved as Mugimba.csv (reduced to ~450–500 rows after cleaning).
Post-cleaning verification confirmed no remaining invalid zeros and significantly reduced variance in extreme values.

3. Feature Engineering, Modelling and Prediction (Part B)
Feature Selection
Selected the top correlated features based on Pearson correlation with the target:
Pregnancies, Glucose, BloodPressure, SkinThickness, BMI, DiabetesPedigreeFunction, Age.
Models Developed
Two classification algorithms were implemented and compared:

1.	Logistic Regression (tuned)
2.	Random Forest Classifier


Performance Comparison (on 20% hold-out test set):

•	Logistic Regression (best model): Accuracy = 67.16%, F1 Score = 50.00%, Precision = 45.83%, Recall = 55.00%.
•	Confusion Matrix:
[[34  13]
[ 9  11]]

Saved Model
Best model saved as Mugimba_best_model.pkl using joblib.
Predictions
Generated on the hold-out test set and saved as Mugimba_predictions.csv.
Interpretation of Performance
The model performs reasonably well given the small dataset and class imbalance (~65% no-diabetes). Main sources of error are class imbalance and noise from the original zero values. In healthcare, false negatives are critical (missed diabetes cases could lead to complications); therefore, the model should be used as a risk-scoring support tool with a lowered decision threshold to prioritise recall.
